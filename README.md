# PCFILTER QA Internship

지란지교에서 8주 동안 QA 인턴으로 근무하며 수행한 PCFILTER 테스트 경험을 공개 가능한 형태로 재구성한 저장소입니다.

PCFILTER는 실제 배포되어 사용되는 엔드포인트 보안 제품이지만, 이 저장소에는 회사 내부 정보가 포함되지 않도록 모든 내용을 포트폴리오용 샘플로 다시 작성했습니다. 내부 URL, 고객사명, 실제 티켓, 인증정보, 로그 원문, 빌드 파일, 사내 화면 스크린샷은 포함하지 않습니다.

## 이 저장소에서 보여주려는 것

- 개발자들이 유지보수하거나 추가 개발한 기능 변경점에 대해 QA 검증을 수행한 방식
- 변경된 기능이 기존 사용자 흐름, 관리자 정책, 로그/리포트와 충돌하지 않는지 확인한 과정
- 보안 제품을 사용자 화면, 정책 조건, 기록 데이터 관점에서 이해한 방식
- 이슈 설명을 재현 가능한 QA 시나리오로 바꾸는 과정
- 조건, 기대 결과, 실제 결과, 증거를 분리해 테스트 기록을 남기는 방식
- 반복되는 테스트 문서 작성 업무를 작은 자동화 샘플로 개선한 흐름
- 회사와 고객 정보를 보호하면서 실무 경험을 공개 문서로 정리하는 기준

## 인턴십 업무 범위

인턴 기간 동안 데스크톱 보안 에이전트와 관련 관리자 설정을 중심으로 QA 업무를 수행했습니다. 개발자들이 유지보수하거나 추가 개발한 기능 변경 사항에 대해 사용자 화면에서 보이는 동작, 정책 반영 여부, 이슈 재현 조건, 로그 및 리포트 일관성을 확인하고 개발자와 QA 리뷰어가 따라갈 수 있는 형태로 결과를 기록하는 데 집중했습니다.

공식 제품 소개에서 공개된 PCFILTER의 기능 범위는 DLP 정보유출방지, 개인정보보호, 취약점 점검, IT 자산관리입니다. 이 저장소에서는 그중 공개 가능한 수준의 기능명을 사용해 QA 관점을 설명합니다.

대표적으로 확인한 영역은 다음과 같습니다.

- 개인정보 검사 흐름: 전체/선택/간편/예약/실시간 검사 결과 확인
- 개인정보 보호조치: 암호화, 삭제, 격리, 파일 보유 관련 동작 확인
- DLP 정보유출방지: 매체, 출력물, 파일 첨부, 공유폴더, 소프트웨어 실행 제어 시나리오 확인
- 관리자 정책 반영: 정책 변경 후 데스크톱 에이전트에 동작이 반영되는지 확인
- 로그 및 리포트: 사용자 화면 결과와 기록 데이터가 일치하는지 확인
- 회귀 테스트: 유지보수 및 추가 개발 변경점이 기존 기능에 영향을 주지 않는지 확인

## 테스트한 변경 유형

개발자들이 제품을 유지보수하거나 추가 개발할 때 QA 관점에서는 아래 내용을 중점적으로 확인했습니다.

- 수정된 기능이 요구사항대로 동작하는지 확인
- 새로 추가된 옵션이나 정책이 사용자 화면에 올바르게 반영되는지 확인
- 기존 검사, 차단, 보호조치 흐름이 깨지지 않는지 확인
- 예외 조건에서 안내 문구와 실패 처리가 자연스러운지 확인
- 결과가 로그, 리포트, 관리자 화면에 일관되게 남는지 확인
- 같은 조건에서 재현 가능한지 반복 테스트로 확인

## Work Sample: QA 가이드 자동화

반복적으로 발생한 업무 중 하나는 이슈 설명을 읽고 테스트 가이드 초안으로 옮기는 일이었습니다. 이 저장소의 샘플 코드는 그 아이디어를 공개 가능한 작은 예제로 재구성한 것입니다.

```text
이슈 요약과 설명
  -> 키워드 기반 QA 대상 선택
  -> 시나리오 초안 생성
  -> 템플릿 안전 검증
  -> 사람이 검토할 수 있는 테스트 가이드 출력
```

자동화 샘플은 실제 Jira, Notion, 내부 제품 데이터를 사용하지 않습니다. 설계 방식과 로직 흐름만 보여주기 위한 예제입니다.

## 배운 점

QA는 단순히 버튼이 동작하는지 확인하는 일이 아니라, 사용자 행동과 정책 조건, 로그, 증거를 하나의 재현 가능한 이야기로 연결하는 과정이라는 것을 배웠습니다. 이 인턴십을 통해 테스트 기록을 더 명확하게 쓰는 법, 반복 업무를 줄이는 법, 실무 경험을 외부에 설명할 때 민감한 정보를 보호하는 법을 익혔습니다.
