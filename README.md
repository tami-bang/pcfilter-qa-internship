# PCFILTER QA Internship

지란지교 QA 인턴십에서 PCFILTER 제품 변경점을 테스트하며 배운 내용을 정리한 공개용 Work Sample입니다.

PCFILTER는 기업 PC의 개인정보보호와 정보유출방지를 다루는 보안 제품입니다. 인턴 기간 동안 저는 개발자들이 유지보수하거나 추가 개발한 변경점이 실제 사용자 흐름에서 잘 동작하는지 확인했습니다.

단순히 화면을 눌러보는 데서 끝내지 않고, 변경된 기능이 정책과 맞게 반영되는지, 기존 기능을 깨뜨리지 않는지, 결과가 로그와 리포트에도 일관되게 남는지 함께 보았습니다. 이 저장소는 그 경험을 회사 내부 정보 없이 다시 구성한 기록입니다.

## 테스트한 것

개발 변경점은 하나의 화면에서만 끝나지 않았습니다. 예를 들어 정책이 바뀌면 데스크톱 에이전트의 메뉴, 검사 결과, 사용자 안내 문구, 로그 기록까지 함께 확인해야 했습니다.

제가 주로 확인한 흐름은 다음과 같습니다.

- 개인정보 검사: 전체 검사, 선택 검사, 간편 검사, 예약 검사, 실시간 검사
- 개인정보 보호조치: 암호화, 삭제, 격리, 파일 보유 현황
- 정보유출방지: 매체, 출력물, 파일 첨부, 공유폴더, 소프트웨어 실행 제어
- 정책 반영: 관리자 설정 변경 후 에이전트 동작 확인
- 기록 확인: 사용자 화면 결과와 로그/리포트의 일관성 확인
- 회귀 테스트: 수정된 기능이 기존 흐름에 영향을 주지 않는지 확인

## 일한 방식

테스트할 변경점을 받으면 먼저 어떤 기능과 정책에 영향을 주는지 정리했습니다. 그다음 사용자가 실제로 지나갈 화면 흐름을 기준으로 시나리오를 만들고, 기대 결과와 실제 결과를 비교했습니다.

결과를 남길 때는 개발자가 다시 확인할 수 있도록 재현 조건, 수행 단계, 실제 결과, 증거, 추가 확인이 필요한 부분을 분리해 적었습니다. 같은 실수를 반복하지 않기 위해 자주 쓰는 화면 경로와 테스트 관점도 문서로 모았습니다.

## 남긴 산출물

### 화면 네비게이션

PCFILTER는 사용자 화면, 관리자 정책, 로그/리포트가 함께 연결되는 제품이라 기능을 확인할 때 같은 경로를 반복해서 찾는 일이 많았습니다. 그래서 화면 경로를 단순 목록으로 두지 않고, “정책이 바뀌면 어디를 확인해야 하는가”, “검사 결과가 이상하면 어떤 기록을 비교해야 하는가”처럼 테스트 목적별로 다시 정리했습니다.

이 문서는 새 테스트를 시작할 때 확인 범위를 빠르게 잡고, 같은 흐름을 다시 찾는 시간을 줄이기 위한 기준이었습니다.

### 테스트 가이드 자동화

테스트 문서 초안을 매번 처음부터 작성하는 일도 반복되었습니다. 이슈 설명에서 테스트 대상과 시나리오 후보를 뽑고, 사람이 이미 작성한 문서는 덮어쓰지 않도록 보호하는 작은 자동화 흐름을 만들었습니다.

```text
이슈 요약과 설명
  -> QA 대상 추론
  -> 시나리오 초안 생성
  -> 템플릿 안전 검증
  -> 사람이 검토할 수 있는 테스트 가이드 출력
```

자동화의 목적은 QA 판단을 대신하는 것이 아니라, 반복되는 초안 작성 시간을 줄이고 사람이 더 중요한 재현 조건과 결과 판단에 집중할 수 있게 하는 것이었습니다.
