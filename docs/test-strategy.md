# 테스트 전략 샘플

## 목표

개발자들이 유지보수하거나 추가 개발한 PCFILTER 기능 변경점이 사용자 화면, 관리자 정책, 로그/리포트 흐름에서 요구사항대로 동작하는지 검증합니다.

PCFILTER는 공개 제품 소개 기준으로 DLP 정보유출방지, 개인정보보호, 취약점 점검, IT 자산관리 기능을 제공하는 보안 솔루션입니다. QA 문서에서는 공개된 기능 범위 안에서만 테스트 대상을 설명합니다.

## QA 관점

테스트는 네 가지 질문을 기준으로 정리합니다.

1. 어떤 사용자 행동이 동작을 발생시키는가?
2. 어떤 정책이나 조건이 해당 동작에 영향을 주는가?
3. 사용자는 어떤 결과를 보게 되는가?
4. 유지보수 또는 추가 개발된 변경점이 기존 흐름을 깨뜨리지 않는가?
5. 그 결과를 증명할 수 있는 증거는 무엇인가?

## 테스트 레이어

| 레이어 | 확인할 내용 | 예시 증거 |
|---|---|---|
| 사용자 흐름 | 메뉴, 버튼, 팝업, 결과 메시지 | 화면 메모 또는 재현 단계 |
| 정책 | 관련 설정의 ON/OFF 상태 | 정책 상태 요약 |
| 클라이언트 동작 | 검사, 차단, 암호화, 삭제, 허용 결과 | 결과 메시지와 시간 |
| 기록 | 로그, 리포트, 관리자 이력 일관성 | 민감정보 제거된 로그/리포트 요약 |

## 변경점 검증 범위

| 변경 유형 | QA 확인 내용 |
|---|---|
| 기능 유지보수 | 수정된 동작이 요구사항대로 재현되는지 확인 |
| 추가 개발 | 새 옵션, 메뉴, 정책, 안내 문구가 사용자 흐름에 자연스럽게 연결되는지 확인 |
| 정책 변경 | 관리자 설정 변경 후 데스크톱 에이전트 동작이 일치하는지 확인 |
| 기록 변경 | 사용자 화면 결과와 로그/리포트 기록이 같은 의미로 남는지 확인 |
| 회귀 위험 | 기존 검사, 차단, 보호조치 흐름이 영향을 받지 않는지 확인 |

## 시나리오 설계

각 시나리오는 아래 항목을 포함합니다.

- 사전 조건
- 테스트 데이터
- 수행 단계
- 기대 결과
- 실제 결과
- 증거
- 후속 판단

## 회귀 테스트 우선순위

회귀 테스트에서는 다음 항목을 우선 확인합니다.

- 변경된 정책의 영향을 받을 수 있는 기존 흐름
- 기능 비활성화, 빈 결과, 권한 누락 같은 경계 상태
- 데스크톱 에이전트, 관리자 설정, 리포트 간 결과 일관성
- 비기술 사용자가 오해할 수 있는 사용자 안내 문구
- 개발 변경점과 직접 관련 없어 보여도 같은 정책 또는 같은 기록 데이터를 공유하는 기능

## 종료 기준

아래 조건을 만족하면 테스트를 종료할 수 있습니다.

- 재현 경로가 명확하다
- 기대 결과와 실제 결과가 비교되어 있다
- 증거가 첨부되었거나 요약되어 있다
- 남은 위험이나 확인 필요 사항이 기록되어 있다
- 공개 문서에 민감정보가 남아 있지 않다
