# 화면 네비게이션 정리

PCFILTER 테스트를 하면서 가장 자주 부딪힌 문제는 “무엇을 확인해야 하는지”보다 “어디에서 확인해야 하는지”였습니다.

하나의 기능 변경도 사용자 화면, 관리자 정책, 로그/리포트에 함께 영향을 줄 수 있었습니다. 그래서 화면 경로를 기능명 기준으로만 모으는 대신, QA가 실제로 던지는 질문을 기준으로 다시 정리했습니다.

이 문서는 Notion 페이지로 작성했고, 같은 페이지 안에 QA 작업 환경을 구성할 수 있는 스타터 키트 zip도 함께 올려두었습니다. 화면 네비게이션과 MCP 설정을 연결해 자연어 질문으로도 필요한 화면 경로와 확인 절차를 찾을 수 있게 만드는 것이 목표였습니다.

## 왜 필요했나

- 같은 화면 경로를 매번 다시 찾는 시간이 반복됨
- 정책 설정 위치와 사용자 화면 결과가 떨어져 있어 흐름을 놓치기 쉬움
- 기능별 테스트 지식이 사람의 기억에 의존하기 쉬움
- 신규 테스트를 시작할 때 확인 범위를 빠르게 잡을 기준이 필요함

## 어떻게 정리했나

화면 네비게이션은 단순한 메뉴 목록이 아니라, 테스트 목적에서 출발하도록 구성했습니다.

예시:

| QA 질문 | 확인 흐름 |
|---|---|
| 정책 변경이 사용자 화면에 반영됐는가? | 관리자 설정 확인 -> 에이전트 정책 갱신 -> 사용자 화면 동작 확인 |
| 검사 결과가 정상적으로 남았는가? | 검사 실행 -> 결과 화면 확인 -> 로그/리포트 기록 비교 |
| 보호조치가 의도대로 동작했는가? | 테스트 파일 준비 -> 보호조치 실행 -> 파일 상태와 사용자 안내 확인 |
| 정보유출방지 정책이 적용됐는가? | 정책 상태 확인 -> 매체/출력물/첨부 흐름 실행 -> 허용/차단 결과 확인 |

## 정리한 범위

- 개인정보 검사 흐름
- 암호화, 삭제, 격리 등 보호조치 흐름
- 매체, 출력물, 파일 첨부, 공유폴더, 소프트웨어 실행 제어 흐름
- 정책 반영 확인 흐름
- 로그 및 리포트 비교 흐름

## 남긴 가치

화면 네비게이션은 테스트 속도를 높이기 위한 문서이기도 했지만, 더 중요하게는 제품을 이해하는 기준이었습니다. 어떤 기능을 테스트할 때 사용자 화면만 볼지, 정책도 확인할지, 기록 데이터까지 비교할지 판단하는 데 도움을 주었습니다.

또한 Codex + MCP 환경에서 이 문서와 기준 파일을 함께 참고하도록 구성해, “무엇을 어디에서 어떻게 확인할지”를 자연어로 물어볼 수 있는 시작점을 만들었습니다.

이 저장소에는 실제 내부 화면 경로와 첨부 zip 원본을 그대로 공개하지 않고, 구조와 작성 방식만 공개 가능한 형태로 재구성했습니다.
