# 2026.06.16 작업 기록

이 문서는 `pcfilter-qa-internship` 저장소를 공개 포트폴리오용 Work Sample로 정리한 작업 기록입니다.

## 작업 배경

PCFILTER는 실제 배포되어 사용되는 보안 제품이므로, 인턴십 경험을 공개할 때 제품명과 공개 기능 범위는 사용할 수 있지만 내부 운영 정보는 제외해야 합니다.

오늘 작업의 목표는 아래와 같았습니다.

- PCFILTER QA 인턴 경험을 별도 GitHub 저장소로 정리
- 개발자들이 유지보수 및 추가 개발한 변경점에 대해 QA 테스트를 수행했다는 점을 명확히 표현
- 공식 제품 소개 페이지에 공개된 기능 범위 안에서 테스트 대상을 설명
- 내부 Jira, Notion, 고객사, 로그, 빌드 파일, 계정 정보가 노출되지 않도록 문서화
- GitHub 프로필과 포트폴리오 케이스스터디에서 이 저장소를 연결

## 반영한 내용

### 1. 공개 저장소 생성

`pcfilter-qa-internship` 저장소를 생성하고, 공개 가능한 QA Work Sample 구조로 구성했습니다.

구성한 문서:

- `README.md`: 인턴십 개요, 테스트 범위, 자동화 샘플 소개
- `SECURITY_AND_PRIVACY.md`: 공개 가능/불가 기준
- `docs/test-strategy.md`: 개발 변경점 검증 중심 테스트 전략
- `docs/checklist-sample.md`: QA 체크리스트 샘플
- `docs/bug-report-sample.md`: 익명화 버그 리포트 샘플

### 2. 인턴십 기간 명시

인턴십 기간을 `2026.04.27 ~ 2026.06.19`로 정리했습니다.

### 3. 테스트 관점 재정리

처음에는 자동화 도구 제작 경험이 앞에 보였지만, 포트폴리오에서 더 중요한 메시지는 실제 제품 변경점에 대한 QA 검증 경험이라고 판단했습니다.

따라서 문구를 아래 방향으로 수정했습니다.

- 개발자 유지보수 및 추가 개발 변경점 검증
- 사용자 화면 동작 확인
- 관리자 정책 반영 확인
- 로그 및 리포트 일관성 확인
- 기존 기능에 대한 회귀 테스트

### 4. 공식 제품 기능 범위 반영

공식 PCFILTER 제품 소개 페이지에서 공개된 범위를 기준으로 기능명을 정리했습니다.

공개 문서에 사용한 표현:

- DLP 정보유출방지
- 개인정보보호
- 취약점 점검
- IT 자산관리
- 개인정보 검사
- 개인정보 보호조치
- 매체, 출력물, 파일 첨부, 공유폴더, 소프트웨어 실행 제어

### 5. 자동화 샘플 포함

반복적인 테스트 문서 작성 흐름을 설명하기 위해 작은 Node.js 샘플을 포함했습니다.

샘플 코드의 목적:

- 이슈 설명에서 QA 시나리오 후보를 생성
- 기존 템플릿에 사용자 작성 내용이 있을 때 자동 수정을 차단
- 실제 내부 API나 데이터 없이 자동화 설계 방식을 설명

## 오늘 정리한 최종 메시지

이 저장소는 PCFILTER를 개발했다는 기록이 아니라, 개발자들이 유지보수 및 추가 개발한 PCFILTER 변경점에 대해 QA 인턴으로서 기능 확인, 정책 반영 확인, 로그/리포트 일관성 확인, 회귀 테스트를 수행한 경험을 공개 가능한 형태로 정리한 자료입니다.
