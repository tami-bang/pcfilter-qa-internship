# Codex + MCP 스타터 키트 정리

화면 네비게이션 문서는 Notion 페이지로 만들고, 그 안에 QA 작업 환경을 빠르게 구성할 수 있는 스타터 키트 zip을 함께 올려두었습니다.

이 키트의 목적은 단순히 파일을 묶어두는 것이 아니었습니다. 테스트할 때 필요한 화면 경로, Notion/Jira 문서, 자동화 스크립트, MCP 설정을 한 흐름으로 연결해 자연어 질문으로도 업무 절차를 찾을 수 있게 만드는 것이었습니다.

## 왜 만들었나

- 새 테스트를 시작할 때 어떤 문서와 화면을 봐야 하는지 찾는 시간이 길었음
- 화면 경로, 테스트 가이드, 자동화 스크립트, MCP 설정이 흩어져 있었음
- Codex를 사용하더라도 기준 문서가 없으면 답변이 실제 제품 화면과 어긋날 수 있음
- 신입 QA가 설치부터 질문 방식까지 따라갈 수 있는 시작점이 필요했음

## 포함한 구성

공개 저장소에는 내부 zip 원본을 올리지 않고, 구성 의도만 정리합니다.

| 구성 | 역할 |
|---|---|
| 시작 가이드 | 설치, 인증정보 입력, MCP 연결, 테스트 시작 흐름 안내 |
| 화면 경로 기준 문서 | PCFILTER, 관리자 페이지, Tools의 기능별 확인 위치 정리 |
| MCP 설정 예시 | Notion, Jira, Codex가 같은 작업 맥락을 볼 수 있도록 연결 |
| 테스트 가이드 자동화 스크립트 | 이슈/문서 내용을 테스트 시나리오 초안으로 변환 |
| 템플릿 보호 로직 | 사람이 작성한 문서를 자동화가 덮어쓰지 않도록 검증 |
| 테스트 코드 | 파싱, 메타데이터 추출, 블록 생성, 보호 로직 검증 |

## 자연어 질문 흐름

스타터 키트의 핵심은 “파일을 압축해서 전달했다”가 아니라, 자연어 질문을 실제 QA 행동으로 연결한 점입니다.

예시 질문:

```text
개인정보 전체검사는 어디서 실행하나요?
USB 차단 정책은 어디서 설정하나요?
정책을 바꾼 뒤 어떤 순서로 확인해야 하나요?
검사 결과가 이상하면 어떤 로그나 리포트를 비교해야 하나요?
```

Codex는 화면 경로 기준 문서와 Notion/Jira 문맥을 함께 참고해, 어디에서 무엇을 어떻게 확인해야 하는지 안내하는 역할을 합니다.

## 안전 기준

스타터 키트에는 실제 업무에 필요한 설정 예시가 있었지만, 공개 포트폴리오에서는 아래 내용을 제외했습니다.

- 내부 Notion 페이지 URL
- Jira 프로젝트 경로와 실제 티켓 번호
- 관리자 페이지 주소
- 계정명, 인증 코드, 토큰, 비밀번호
- 테스트 빌드 파일과 내부 첨부 원본
- 사내 화면 캡처

## 남긴 가치

이 작업은 QA 문서를 단순히 보기 좋게 정리한 것이 아니라, 테스트 흐름을 재사용 가능한 작업 환경으로 묶은 경험이었습니다. 화면 네비게이션은 “어디서 확인하는가”를 잡아주고, MCP 연결은 “자연어로 물어도 어떤 문맥을 봐야 하는가”를 이어주었습니다.
