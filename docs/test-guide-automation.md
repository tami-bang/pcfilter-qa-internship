# 테스트 가이드 자동화 정리

반복되는 테스트 문서 작성 시간을 줄이기 위해 테스트 가이드 자동화 흐름을 정리했습니다.

이 자동화는 QA 판단을 대신하기 위한 도구가 아니었습니다. 이슈 설명을 읽고 테스트 초안을 만드는 반복 작업을 줄이고, 사람은 재현 조건과 결과 판단에 더 집중할 수 있게 만드는 보조 도구였습니다.

## 왜 필요했나

- 이슈마다 테스트 목적, 조건, 시나리오를 문서로 옮기는 일이 반복됨
- 비슷한 기능은 매번 비슷한 테스트 항목을 다시 작성하게 됨
- 자동 작성 기능이 기존 사용자 내용을 덮어쓰면 안 됨
- 테스트 문서가 일정한 구조를 갖춰야 개발자와 QA 리뷰어가 다시 보기 쉬움

## 자동화 흐름

```text
이슈 요약과 설명
  -> 키워드 기반 QA 대상 추론
  -> 테스트 시나리오 초안 생성
  -> 문서 템플릿 상태 확인
  -> 사용자 작성 내용 보호
  -> 사람이 검토할 수 있는 가이드 출력
```

## 핵심 로직

### 이슈 파싱

이슈 요약과 설명에서 테스트 대상이 될 만한 키워드를 찾고, 관련된 사용자 흐름을 시나리오 후보로 변환했습니다.

예시:

| 이슈 내용에서 보이는 단서 | 생성되는 테스트 관점 |
|---|---|
| 간편 검사, 우클릭 메뉴 | 파일 단위 검사 흐름 확인 |
| 정책, 동기화, 반영 안 됨 | 관리자 설정과 에이전트 동작 비교 |
| 로그, 리포트, 내보내기 | 사용자 화면 결과와 기록 데이터 비교 |

### 템플릿 보호

자동화가 이미 작성된 문서를 망가뜨리지 않도록 보호 조건을 두었습니다.

- 완료 마커가 있으면 다시 실행하지 않음
- 사용자 작성 텍스트가 있으면 자동 수정을 막음
- 이미지, 파일, 표 같은 사용자 소유 블록이 있으면 중단
- 필수 섹션이 없으면 자동 작성하지 않음

## 저장소 샘플

이 저장소에는 실제 내부 API나 업무 데이터를 넣지 않고, 설계 방식을 보여주는 작은 Node.js 샘플만 포함했습니다.

- `src/issue-parser.js`: 이슈 설명에서 테스트 시나리오 후보 생성
- `src/template-guard.js`: 사용자 작성 내용 보호
- `test/automation.test.js`: 파싱과 보호 로직 검증

실행:

```text
npm test
```

## 남긴 가치

테스트 가이드 자동화는 문서를 “자동으로 완성하는 도구”라기보다, QA가 반복해서 쓰는 구조를 안정적으로 준비해주는 도구였습니다. 덕분에 문서 초안 작성보다 재현 조건, 기대 결과, 실제 결과를 더 정확히 보는 데 시간을 쓸 수 있었습니다.
